MyALM

资产负债管理系统学习知识库

本知识库用于系统学习银行 ALM。当前范围聚焦流动性风险与利率风险，不包含 FTP。

侧边栏由 docs/sidebar.js 统一生成。后续新增模块时，只需要维护一个目录配置。

一、模块划分

基础认知

解释 ALM 系统定位、边界、数据来源，以及与核心、风险、管理会计、FTP 的关系。

共同基础

覆盖现金流展开、期限分桶、Gap 分析等流动性风险和利率风险共用的基础能力。

流动性风险

关注未来资金流入流出、期限缺口、优质流动性资产、LCR、NSFR 和压力情景。

利率风险

关注利率变化对银行账簿经济价值和净利息收入的影响，包括 IRRBB、EVE、NII。

模型与场景

沉淀提前还款、提前支取、无到期存款、利率冲击、流动性压力等模型假设。

系统实现

整理账户级数据模型、参数表、批处理流程、数据质量校验和性能优化方法。

二、核心学习路径

阶段学习重点对应模块
1理解 ALM 解决什么问题ALM 总览
2理解账户如何生成未来现金流现金流展开
3区分期限缺口与重定价缺口Gap 分析
4学习流动性风险指标流动性风险、LCR、NSFR
5学习利率风险指标利率风险、IRRBB、EVE、NII
6沉淀系统开发视角数据模型、批处理与性能

三、当前不纳入范围

FTP 在很多银行作为独立系统存在，主要用于内部资金转移定价和经营绩效评价。本知识库不编写 FTP 计算模块，只在说明系统边界时提及。
