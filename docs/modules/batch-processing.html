MyALM

批处理与性能

ALM 批处理通常围绕数据加载、现金流展开、情景计算、指标汇总、结果校验和报表落库展开。

一、典型批处理链路

数据加载 → 数据校验 → 现金流展开 → 情景计算 → 指标汇总 → 结果校验 → 报表输出

二、常见性能压力点

环节压力来源关注点
数据加载账户级数据量大分区、批量加载、增量策略
现金流展开一笔长期资产可展开为多期现金流控制中间表规模、并行策略
情景计算多个情景重复计算情景编号、复用基础现金流
指标汇总多维度聚合机构、产品、币种、期限桶分层汇总
结果校验口径复杂总分核对、余额守恒、异常账户清单

三、开发关注点

  • 大表要明确分区键，常见选择是批次号、日期、情景编号。
  • 中间表命名要能区分模块、步骤和批次。
  • 现金流展开前后要做本金、余额和条数校验。
  • 复杂计算优先拆分步骤，便于定位问题和重跑。
  • 结果表保留来源批次和参数版本。
