ALM 总览

资产负债管理系统关注银行资产、负债和表外项目在期限、利率、现金流上的结构性风险。

一、系统定位

ALM 不是账务系统，也不是单纯报表系统。它以账户级资产负债数据为基础，生成未来现金流，并在不同期限桶、利率情景和压力情景下计算风险指标。

二、核心范围

模块核心问题典型输出
流动性风险未来有没有足够资金应对流出现金流缺口、LCR、NSFR
利率风险利率变化会怎样影响价值和利润重定价缺口、EVE、NII
压力测试极端但可能发生的情景下风险如何变化压力结果、预警指标

三、与 FTP 的边界

FTP 主要服务于内部资金定价和经营绩效评价。本知识库将 FTP 作为独立系统看待，不纳入 ALM 主线。

四、开发视角

开发工作通常集中在账户级数据加工、现金流展开、参数维护、指标落库、批处理调度、数据核对和报表展示。
