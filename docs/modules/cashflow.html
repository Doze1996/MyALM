现金流展开

现金流展开是 ALM 的共同基础，用于把账户、合同或组合拆解为未来各期本金、利息和行为调整后的现金流。

一、业务问题

银行资产负债表上的余额只是某个时点的存量。ALM 更关心这些存量未来在什么时间形成资金流入、资金流出、重定价或缺口暴露。

现金流展开的核心任务，是把账户级或组合级数据按照合同规则、产品特征、客户行为假设和情景参数，转换成未来日期或期限桶上的现金流明细。

问题现金流展开要回答的内容
未来有没有钱收回来资产本金回收、贷款还款、债券兑付、同业资产到期。
未来有没有钱付出去存款支取、同业负债到期、发行债务兑付、利息支付。
收付时间是否错配按日、月、季或监管期限桶统计流入流出，形成期限缺口。
利率变化会影响多少收益在重定价日、到期日或浮动利率调整日重新计算未来利息。
压力情景下是否还成立叠加提前还款、提前支取、存款流失、续作下降等行为假设。

二、常见输入

数据说明开发关注点
账户余额当前本金、账面余额或摊余成本。要明确使用本金余额还是会计账面余额。
起息日 / 到期日决定合同现金流的起止范围。活期、循环额度等产品可能没有明确到期日。
利率与计息规则包括固定利率、浮动利率、利率调整频率、计息基础。需要区分利息现金流和重定价现金流。
还款方式到期一次还本、等额本息、等额本金、计划还款、不规则还款。不同还款方式不能套用同一套本金分摊逻辑。
付息频率按月、按季、按半年、按年或到期一次付息。本金日期和利息日期可能不同。
行为假设提前还款、提前支取、活期留存、展期、续作、违约回收。行为模型一般按产品、客户、机构、币种或情景分层。
情景参数基准情景、利率上行情景、流动性压力情景、监管压力情景。所有输出都应保留情景编号，避免结果混用。

三、现金流类型

现金流展开不能只生成一个金额字段。至少要区分本金、利息、行为调整和表外转实表等类型，否则后续期限缺口、LCR、NSFR、EVE、NII 很难复用。

现金流类型方向典型来源说明
本金流入流入贷款还本、债券兑付、同业资产到期。资产端本金回收，通常改善流动性。
本金流出流出存款支取、同业负债到期、发行债务兑付。负债端本金偿付，通常形成资金压力。
利息流入流入贷款利息、债券票息、同业资产利息。进入 NII 计算，也影响未来现金收入。
利息流出流出存款利息、同业负债利息、发行债务利息。进入 NII 计算，也影响未来现金支出。
提前还款流入住房贷款、消费贷、经营贷。合同外本金提前回收，会缩短资产久期。
提前支取流出定期存款、大额存单。合同外资金提前流出，会加大短期流动性压力。
续作 / 展期调整项同业业务、存款、贷款。到期后不完全流出或流入，而是部分形成新余额。
表外提款流出授信额度、信用卡额度、保函承诺。压力情景下可能转化为表内资金流出。

四、按产品区分展开逻辑

现金流展开的关键不是写一个统一公式，而是先识别产品类型。不同产品的合同约束和客户行为完全不同。

1. 到期一次还本类

这类产品的本金在到期日一次性流入或流出，期间只按约定频率产生利息现金流。

项目说明
典型产品同业拆放、同业存单、部分债券、定期存款、发行债务。
本金逻辑本金金额全部落在到期日。
利息逻辑按付息周期生成利息，若到期一次付息，则利息也落在到期日。
校验重点展开后本金合计应等于当前本金余额。
 
到期本金现金流 = 当前本金余额
期间利息现金流 = 本金余额 × 合同利率 × 计息天数 / 年基准天数

2. 分期还款类

分期还款类产品每期同时产生本金和利息。常见于住房贷款、消费贷、经营贷、租赁资产等。

还款方式本金计算利息计算特点
等额本息每期还款额固定，本金逐期增加。按剩余本金计息，利息逐期下降。前期利息多、本金少。
等额本金本金按期均摊。按剩余本金计息，利息逐期下降。每期总还款额递减。
计划还款按还款计划表直接取数。可按计划表或计息规则计算。适合对公贷款、不规则还款。
气球贷期间少量还本，到期大额还本。按剩余本金计息。到期日存在集中本金流。
 
每期利息 = 期初剩余本金 × 本期利率 × 本期计息天数 / 年基准天数
每期本金 = 每期还款额 - 每期利息
期末剩余本金 = 期初剩余本金 - 每期本金 - 提前还款本金

3. 活期和无明确到期类

活期存款、保证金存款、部分结算性存款没有合同到期日。若全部放在隔夜，会严重高估短期流出；若全部视为长期稳定，也会低估压力风险。

分层处理方式说明
易流失余额分布到短期限桶。例如隔夜、7 天、1 个月内。
稳定余额按行为期限分布到较长期限桶。需要根据历史留存、客户类型、利率敏感度估计。
核心存款可在利率风险中按较长期限处理。口径依赖银行内部模型，具体参数应待核对。

这类产品通常使用无到期存款模型（NMD）。模型结果不是改变真实合同，而是为了 ALM 计量把余额按行为期限重新分布。

4. 循环额度类

信用卡、循环贷款和授信额度的余额会反复提款和还款。合同到期日不能完全代表未来现金流。

项目处理方式
已提款余额按最低还款、历史还款率、账龄迁移或合同计划展开。
未提款额度在压力情景下按提款率转换为未来资金流出。
循环使用通过余额衰减率、续借率或使用率模型估计。
开发处理通常不按单笔交易逐日模拟，而是按账户或组合参数展开。

5. 可提前还款资产

住房贷款等长期资产即使合同期限很长，也可能因客户置换贷款、卖房、利率下降或资金宽裕而提前还款。

本期现金流 = 正常还款本金 + 正常还款利息 + 提前还款本金
提前还款本金 = 期初剩余本金 × 提前还款率

提前还款率可以按月度提前还款率、年化提前还款率或分层参数维护。参数通常会受利率变化、贷款账龄、客户类型、地区和产品政策影响。

6. 可提前支取负债

定期存款、大额存单等负债合同上有到期日，但客户可能提前支取。压力情景下提前支取比例通常会上升。

现金流处理说明
合同到期部分仍按原到期日形成本金流出和利息流出。
提前支取部分按提前支取模型分布到更早期限桶。
利息调整提前支取后的利息可能按活期或约定规则重算，具体口径需按产品规则待核对。

7. 违约和回收类现金流

对贷款资产，合同现金流不一定能全部收回。ALM 中是否纳入违约、回收和核销，需要结合系统边界确定；如果纳入，通常会对本金和利息现金流做折减或迁移。

处理方式说明
合同现金流法先按合同展开，再在风险系统或减值系统中处理信用风险。
折减现金流法按违约概率、损失率或回收率对未来现金流做调整。
迁移现金流法违约后停止正常利息流入，转入催收、处置或回收现金流。

五、基础计算流程

系统实现上，现金流展开通常分为合同现金流、行为调整、情景调整和汇总分桶四步。

  1. 读取账户余额、合同属性、利率、还款计划和产品参数。
  2. 根据产品类型生成基准合同现金流。
  3. 叠加提前还款、提前支取、续作、存款流失、提款率等行为模型。
  4. 叠加情景参数，例如利率冲击、流动性压力、客户行为恶化。
  5. 按现金流日期、期限桶、币种、机构、产品、情景编号汇总。
  6. 做本金守恒、日期范围、正负方向、重复展开等校验。
步骤输入输出校验点
合同展开账户、合同、利率、还款方式。合同本金和合同利息。本金是否守恒，到期日是否正确。
行为调整提前还款率、提前支取率、留存率、续作率。行为调整后的现金流。调整前后余额是否可解释。
情景加工情景编号、压力参数、利率曲线。不同情景下的现金流版本。基准和压力情景不能覆盖混淆。
期限分桶现金流日期、期限桶规则。按期限桶汇总结果。边界日期是否落桶正确。

六、情景分析

现金流展开不能只计算基准情景。ALM 需要观察不同市场和客户行为假设下，未来现金流结构如何变化。

情景主要假设对现金流的影响
基准情景按当前合同和正常行为参数展开。形成日常管理口径的基础结果。
利率上行情景市场利率上升，客户可能减少提前还款，存款成本上升。资产久期可能拉长，负债利息流出上升。
利率下行情景市场利率下降，按揭贷款提前还款可能上升。资产本金提前回收，再投资收益可能下降。
流动性压力情景批发资金续作下降，存款流失率上升，表外提款增加。短期净现金流可能明显恶化。
特定压力情景房地产、区域、行业或大客户风险事件。相关组合现金流提前、延后或折减。

七、输出结构

常见输出是账户级或汇总级未来现金流表。账户级结果便于追溯，汇总级结果便于指标计算和报表展示。

字段说明
scenario_id情景编号，例如基准、利率上行、流动性压力。
as_of_date数据日期或估值日期。
account_id账户或合同标识，汇总结果可为空或使用组合编号。
org_id / product_id / currency机构、产品、币种等分析维度。
cashflow_date现金流实际发生日期或预测发生日期。
bucket_id期限桶，例如隔夜、7 天、1 个月、3 个月、1 年以上。
cashflow_type本金流入、本金流出、利息流入、利息流出、提前还款等。
amount现金流金额，建议方向和类型明确，不要只依赖正负号表达业务含义。

八、开发关注点

  • 大批量账户展开后数据量会快速膨胀，应区分是否需要账户级明细长期保留。
  • 本金现金流与利息现金流必须分开，不要只保留净额。
  • 合同现金流、行为调整现金流和压力情景现金流应保留来源标识。
  • 所有结果要保留情景编号，避免基准情景和压力情景混淆。
  • 日期处理要统一自然日、工作日、节假日顺延、月末规则和计息天数规则。
  • 要有中间结果校验，例如展开前后本金是否守恒、现金流日期是否超出到期日、同一账户是否重复展开。
  • 对无明确到期、提前支取、提前还款等模型参数，应标明参数版本和生效日期。
