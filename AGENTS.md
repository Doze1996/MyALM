# AGENTS.md

本文件用于约束 AI Agent / Codex / 自动化助手在本仓库中的行为。

## 项目定位

MyALM 是银行资产负债管理系统学习知识库。

重点范围：

- 流动性风险管理
- 利率风险管理
- 现金流展开
- 期限缺口、重定价缺口
- LCR、NSFR
- IRRBB、EVE、NII
- 压力测试、情景模拟
- 数据模型、批处理、数据质量

明确排除：

- FTP 系统实现
- 管理会计绩效分摊
- 与 ALM 无关的通用金融知识堆砌

## 总体原则

1. 优先在现有结构上补充内容，不要随意重构目录。
2. 不要擅自更换技术方案。
3. 文档使用 HTML 编写，除 README 和 AGENTS 外，不新增 Markdown 知识正文。
4. 最终提交物只保留最终内容，不保留讨论过程、调试记录、临时说明。
5. 内容应尽量贴近国内银行 ALM 系统实践。
6. 不确定的监管口径、公式或术语必须标注“不确定”或“待核对”。
7. 不编造来源、法规、产品功能或系统表结构。

## HTML 文档规范

知识正文放在：

```text
docs/modules/
```

每个模块一个 HTML 文件。

页面结构建议：

```html
<section class="doc-section">
  <h1>模块标题</h1>
  <p>模块说明。</p>

  <h2>一、业务问题</h2>
  <p>...</p>

  <h2>二、核心概念</h2>
  <p>...</p>

  <h2>三、数据与计算</h2>
  <p>...</p>

  <h2>四、开发关注点</h2>
  <p>...</p>
</section>
```

## 侧边栏维护规范

侧边栏唯一维护入口：

```text
docs/sidebar.js
```

新增模块时，只允许修改 `SIDEBAR_ITEMS` 数组。

不要在各个 HTML 页面里手写重复侧边栏。

新增项格式：

```js
{
  group: '模块分组',
  title: '页面标题',
  href: './modules/example.html'
}
```

## 内容写作规范

- 中文为主。
- 优先使用表格解释概念差异。
- 对 ALM 术语要区分业务口径，例如：
  - 期限缺口：用于流动性风险；
  - 重定价缺口：用于利率风险；
  - Gap 只是差额概念，必须结合上下文解释。
- 避免把 FTP 混入 ALM 主线。
- 可以说明 FTP 与 ALM 的边界，但不要写 FTP 计算模块。

## 禁止行为

- 不要把本项目改造成 Vue、React、Vite、MkDocs、Docusaurus 等框架项目，除非用户明确要求。
- 不要新增构建链路。
- 不要引入外部 CDN 依赖。
- 不要生成大段空泛金融介绍。
- 不要在 HTML 正文里写“本次修改”“根据你的要求”等对话痕迹。
- 不要删除已有模块，除非用户明确要求。

## 推荐工作流

修改前：

1. 读取相关 HTML 文件；
2. 读取 `docs/sidebar.js`；
3. 判断是否只需补充内容；
4. 优先最小修改。

修改后：

1. 检查链接是否正确；
2. 检查侧边栏是否包含新模块；
3. 检查内容是否仍排除 FTP；
4. 检查页面是否可直接由浏览器打开。
